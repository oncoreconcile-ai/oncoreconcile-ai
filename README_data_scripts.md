# OncoReconcile AI Data Collection Scripts

These scripts support the current data pipeline:

```text
Public oncology knowledge sources
-> downloaded candidate genes / variants
-> human review
-> curated MVP catalogs
-> benchmark generation
-> reconciliation engine
```

## Requirements

```bash
pip install requests pandas
```

`pandas` is not required by the current checked-in scripts, but it is useful for
reviewing the generated CSV files.

## Active Data Inputs

The reconciliation and benchmark workflow currently uses:

```text
data/disease_aliases.json
data/gene_aliases.json
data/gene_variant_catalog.csv
data/benchmark_cases.csv
```

Additional supporting files:

```text
data/disease_gene_catalog.csv
data/raw/raw_gene_alias_candidates.json
data/raw/civic_variant_candidates.csv
```

`data/disease_gene_catalog.csv` is a curated/static disease-to-gene support file
at the moment. It is not generated by the current scripts.

## Run Order

### 1. Download gene aliases from MyGene.info

```bash
python scripts/download_gene_aliases_from_mygene.py
```

Outputs:

```text
data/gene_aliases.json
data/raw/raw_gene_alias_candidates.json
```

### 2. Download candidate variants from CIViC GraphQL

```bash
python scripts/download_variant_candidates_from_civic_graphql.py
```

Outputs:

```text
data/raw/civic_variant_candidates.csv
```

Optional CIViC API key:

```bash
export CIVIC_API_KEY="your_key_here"
python scripts/download_variant_candidates_from_civic_graphql.py
```

### 3. Create MVP-safe gene-variant catalog

For the smaller seed catalog:

```bash
python scripts/create_gene_variant_catalog.py
```

For the expanded catalog used by the current benchmark:

```bash
python scripts/create_expanded_gene_variant_catalog.py
```

Output:

```text
data/gene_variant_catalog.csv
```

### 4. Optional: Create curated catalog from a reviewed file

First create:

```text
data/reviewed_gene_variant_catalog.csv
```

Required columns:

```csv
disease_scope,gene,variant,variant_aliases,alteration_type,source,mvp_status,notes,approved
```

Then run:

```bash
python scripts/create_curated_catalog_from_review.py
```

Output:

```text
data/gene_variant_catalog.csv
```

Only rows with `approved` set to `yes`, `y`, `true`, or `1` are promoted.

### 5. Generate benchmark cases from catalogs

```bash
python scripts/generate_benchmark_cases_from_catalogs.py
```

Inputs:

```text
data/disease_aliases.json
data/gene_aliases.json
data/gene_variant_catalog.csv
```

Output:

```text
data/benchmark_cases.csv
```

The current generated benchmark contains 156 cases.

## Notes

- `data/raw/*` contains candidate data for review.
- `data/disease_aliases.json`, `data/gene_aliases.json`, and `data/gene_variant_catalog.csv` are curated MVP inputs.
- `data/benchmark_cases.csv` is generated from the curated aliases and variant catalog.
- Do not blindly promote downloaded records into production.
- Ambiguous terms like `TRK` should route to `REVIEW_REQUIRED`.
- Unknown terms should route to `CANNOT_RECONCILE`.
